How lane timing works

Controller
Harness orchestration around the main solve loop. Examples: retry, wait, finish, finalize, task updates.
Model
LLM call spans: prompt-in, model-out, and prompt carry-forward effects. Example: one model response that sets up the next tool action.
Workspace
Task-facing work such as reading files, editing code, and ordinary terminal/tool actions. Examples: open file, patch code, run tests.
Runtime
Internal overhead and persistence activity rather than task-solving work. Examples: LocalFileStore and harness plumbing.
Active time
Union of observed wall-clock intervals for a lane, so overlapping events within the same lane are merged instead of double-counted.
Burst
A consecutive same-lane stretch in observed event order, shown as one occupancy bar segment.
Handoff
One lane burst followed by the next lane burst in trace order. The Sankey width shows how often that transition happened.
